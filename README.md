# WhisperX Subtitle Generator

A GUI tool for batch-generating subtitles for large media libraries using your NVIDIA GPU. Transcribes audio with WhisperX and translates using Argos Translate (offline) or Google Translate.

Built mainly for Plex libraries where existing subtitles from tools like Bazarr don't match properly.

---

## Requirements

- Windows 10/11
- Python 3.11 — https://python.org/downloads/release/python-3110/
  - Check **"Add Python to PATH"** during install
- NVIDIA GPU (GTX 1000 series or newer)
- Internet connection for first-time model downloads

---

## Installation

1. Clone or download this repo
2. Double-click **`install.bat`**
3. Wait — PyTorch is ~2.5GB so it takes a while
4. Run with **`run.bat`**

---

## Usage

The tool has two modes, switchable at the top.

### Library Manager (default)

This is the main way to use the tool when managing a large library.

1. Add one or more root folders (e.g. `Z:\media\tv`, `Z:\media\movies`)
2. Click **Scan** — it walks all subfolders and checks subtitle status
3. Each folder shows a status badge:
   - `● NEW` — no subtitles found
   - `✓ DONE` — our subtitles present
   - `▲ CHANGED` — our subtitles exist but the video file is newer
   - `~ EXTERNAL` — subtitles exist but weren't made by this tool (e.g. Bazarr)
   - `◑ MIXED` — some episodes done, some not
4. Use the filter buttons to narrow down, or **Select NEW** / **Select CHANGED**
5. Click **▶ Process Selected** to start

The library state is saved to `library.json` next to the script, so it remembers your roots and scan results between sessions.

**Identifying our subtitles:** Every SRT we generate starts with a comment tag `; generated by whisperx-subgen`. This is how the scanner tells our files apart from Bazarr or other external subs.

### Manual Queue

Switch to Manual Queue mode to process specific folders directly without the library scanner. Add folders, reorder them, scan, and start.

## Settings

**Whisper model** — bigger = more accurate, slower

| Model | VRAM | Speed (RTX 3090) | Quality |
|-------|------|-----------------|---------|
| tiny | ~1GB | ~32x realtime | poor |
| base | ~1.5GB | ~16x realtime | okay |
| small | ~2.5GB | ~8x realtime | good |
| medium | ~5GB | ~4x realtime | very good |
| large-v3 | ~10GB | ~2x realtime | best |

**Compute type** — `float16` for 8GB+ VRAM, `int8_float16` if you have less

**Translation engine**
- `Argos (local, offline)` — fully offline, no rate limits. Downloads a ~100MB language pack on first use, then cached permanently.
- `Google Translate (online)` — better quality but requires internet and may rate-limit on large batches

**Batch size** — how many audio chunks to process at once. Higher = faster but more VRAM. 4-8 is a safe range for most GPUs.

**Scan for lang** — the language code to look for when scanning the library (e.g. `no`, `en`, `sv`). Defaults to `no`.

## Options

- **Skip files that already have a .en.srt / .no.srt** — don't reprocess completed files
- **Also save original-language .srt** — saves raw transcription before translation
- **Delete existing .en.srt / .no.srt** — removes old subtitle files before generating new ones, useful for replacing Bazarr subs
- **Dry run** — lists files without processing

---

## Tips

- Close Chrome and other GPU-heavy apps before long runs
- The first run downloads AI models (~1-5GB depending on model) — subsequent runs are faster
- If the GPU crashes mid-batch, close and reopen the GUI to reset CUDA, then run again — completed files are skipped automatically
- For large backlogs, `small` model is a good tradeoff between speed and quality
- After a batch finishes, the library view automatically refreshes to show updated status

## Troubleshooting

**"python is not recognized"** — Python not in PATH. Reinstall and check "Add to PATH".

**"No CUDA GPU found"** — PyTorch installed without CUDA. Run `install.bat` again.

**CUDA crashes mid-batch** — Close background GPU apps (Chrome, games, OBS), lower batch size.

**Translation not working (Google)** — Check internet, or switch to Argos.

**Subtitles out of sync** — Alignment occasionally fails and falls back to proportional timing. Re-running that file usually fixes it.

**Library shows EXTERNAL for files I made** — Older SRT files generated before this tag was added won't be recognized. Re-run those files and they'll be tagged correctly going forward.
