# WhisperX Subtitle Generator

A GUI tool for batch-generating subtitles for large media libraries using your NVIDIA GPU. Transcribes audio with WhisperX and translates using Argos Translate (offline) or Google Translate. Built for Plex libraries where existing subtitles from tools like Bazarr don't match properly.

---

## Requirements

- Windows 10/11
- Python 3.11 — https://www.python.org/downloads/release/python-3110/
  - During install, check **"Add Python to PATH"**
- NVIDIA GPU (GTX 1000 series or newer)
- ffmpeg (full-shared build) — https://github.com/BtbN/FFmpeg-Builds/releases
  - Extract and add the `bin` folder to your PATH
- Internet connection for first-time model and package downloads

---

## Installation

1. Download or clone this repo
2. Double-click **`install.bat`**
3. Wait for it to finish — PyTorch alone is ~2.5GB
4. Run the tool with **`run.bat`**

The installer handles PyTorch, all Python dependencies, and the Argos translation language pack automatically.

---

## First run

WhisperX downloads the Whisper model on first use (~75MB for `small`, ~1.5GB for `large-v3`). This only happens once — models are cached locally after that.

## Workflow

### 1. Add files via Library Manager

Click **📂 Open Library Manager** to browse your media library.

- Add one or more root folders (e.g. `Z:\media\tv`, `Z:\media\movies`) and click **Scan Library**
- Each folder gets a status badge:
  - `● NEW` — never seen before, no subtitles
  - `○ UNPROCESSED` — seen before, still no subtitles
  - `✓ DONE` — our subtitle exists, file unchanged
  - `▲ CHANGED` — our subtitle exists but the video was modified since
  - `◑ PARTIAL` — some episodes done, some not
  - `~ EXTERNAL` — subtitle exists but not made by this tool
  - `⊕ EMBEDDED` — no external subtitle but the video has embedded subtitle streams
- Expand any folder to see individual episode status
- Use the filter buttons or search box to narrow down
- Select what you want → click **＋ Add to Queue** → library closes

The library state is saved to `library.json` and persists between sessions. On startup it loads from cache — no NAS access until you click Scan.

### 2. Adjust settings

Back on the main window, check your model and translation settings before starting.

### 3. Start processing

Click **▶ Start Processing**. The queue shows each file's status live:
- `▶` running
- `✓` done
- `✗` failed

Use **⏹ Stop After Current** to stop cleanly after the current file finishes.

Library status updates automatically as each file completes.

## Plex compatibility

Subtitle files are saved next to each video using Plex's naming convention:
```
Show - S01E01.en.srt
Show - S01E01.no.srt
```

After processing, do a **Refresh Metadata** on the show/season in Plex so it picks up the new subtitle files. If the video has embedded subtitles, Plex may default to those — go to the episode's subtitle settings and switch to the external SRT.

The tool appends a comment tag `; generated by whisperx-subgen` at the end of every SRT it generates. This is used by the library scanner to identify its own files — it doesn't affect playback.

## Settings

**Whisper model**

| Model | VRAM | Speed (RTX 3090) | Notes |
|-------|------|-----------------|-------|
| tiny | ~1 GB | ~32x realtime | Poor quality |
| base | ~1.5 GB | ~16x realtime | Okay |
| small | ~2.5 GB | ~8x realtime | Good balance |
| medium | ~5 GB | ~4x realtime | Very good |
| large-v3 | ~10 GB | ~2x realtime | Best |

**Compute type**
- `float16` — recommended for 8GB+ VRAM
- `int8_float16` — use if you get out-of-memory errors
- `int8` — CPU fallback, very slow

**Translation engine**
- `Argos (local, offline)` — fully offline after install, no rate limits, good for large batches
- `Google Translate (online)` — better quality but requires internet, may rate-limit on large runs

**Batch size** — audio chunks processed at once. The app auto-caps based on your GPU's VRAM. Start at 4 and increase if stable.

**Sub start offset (s)** — shifts subtitle start time forward to compensate for WhisperX including a small amount of silence before speech. Default 0.25s. If subtitles appear before people speak, increase this value. If too late, decrease it.

**Scan for lang** — language code the library scanner checks for when determining status (e.g. `no`, `en`, `sv`). Should match your target translation language.

## Options

- **Skip if .en.srt / .no.srt exists** — skip files that already have subtitles
- **Save original-language .srt** — saves the raw transcription before translation
- **Delete existing .en.srt / .no.srt** — removes existing subtitle files before generating new ones; useful for replacing Bazarr subs
- **Dry run** — lists files without processing anything

---

## Tips

- Close Chrome, games, and other GPU-heavy apps before long runs
- The library scanner only touches the NAS when you click Scan — drives stay asleep otherwise
- If the GPU crashes, close and reopen the app to reset CUDA, then run again — completed files are skipped
- `small` is a good starting point for large backlogs; use `medium` for better quality
- After a batch, do a Plex metadata refresh on affected shows to pick up the new subtitle files

## Troubleshooting

**"python is not recognized"**
Reinstall Python 3.11 and check "Add Python to PATH".

**"No CUDA GPU found"**
PyTorch was installed without CUDA. Re-run `install.bat`.

**CUDA crash / computer restart**
Close GPU-heavy background apps. Lower batch size. Re-run — completed files are skipped.

**Subtitles appear before speech**
Increase **Sub start offset** (e.g. to 0.5 or 1.0).

**Subtitles appear too late**
Decrease **Sub start offset** toward 0.

**Plex not showing new subtitles**
Right-click the show in Plex → Refresh Metadata. If the video has embedded subs, Plex may prefer those — switch to the external SRT in the playback subtitle menu.

**Transcoder fails in Plex when selecting subtitle**
The SRT file may have been corrupted by a previous failed run. Delete it and regenerate.

**Argos translation not working**
Re-run `install.bat` to reinstall the language pack.

**Library shows EXTERNAL for files I generated before**
Old files don't have the identification tag. Re-run those files to regenerate them.
