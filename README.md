# WhisperX Subtitle Generator

A GUI tool for batch-generating subtitles for large media libraries using your NVIDIA GPU. Transcribes audio with WhisperX and translates using Argos Translate (offline) or Google Translate.

Built mainly for Plex libraries where existing subtitles from tools like Bazarr don't match properly.

---

## Requirements

- Windows 10/11
- Python 3.11 — https://www.python.org/downloads/release/python-3110/
  - During install, check **"Add Python to PATH"**
- NVIDIA GPU (GTX 1000 series or newer)
- Internet connection for first-time model and package downloads

---

## Installation

1. Download or clone this repo
2. Double-click **`install.bat`**
3. Wait for it to finish — PyTorch alone is ~2.5GB
4. Run the tool with **`run.bat`**

That's it. The installer handles PyTorch, all dependencies, and the Argos translation language pack.

---

## First run

On first use, WhisperX will download the Whisper model you've selected (~75MB for `small`, ~1.5GB for `large-v3`). This only happens once — models are cached locally after that.

## Usage

### Library Manager

Click **📂 Open Library Manager** to open the library browser. This is the main way to manage a large collection.

1. Add one or more root folders (e.g. `Z:\media\tv`, `Z:\media\movies`) and click **Scan Library**
2. Each show/movie folder gets a status badge:
   - `● NEW` — never seen before, no subtitles
   - `○ UNPROCESSED` — seen in a previous scan, still no subtitles
   - `✓ DONE` — subtitles generated by this tool, file unchanged
   - `▲ CHANGED` — subtitles exist but the video file was modified since
   - `◑ PARTIAL` — some episodes done, some not
   - `~ EXTERNAL` — subtitles exist but weren't made by this tool (e.g. Bazarr)
3. Expand any folder to see episode-level status
4. Use the filter buttons or search to narrow down
5. Select folders or individual episodes, then click **▶ Process Selected**

The library state is saved to `library.json` next to the script and persists between sessions. On startup it loads from cache — no NAS access until you click Scan.

**How it identifies its own subtitles:** Every SRT this tool generates starts with a hidden comment `; generated by whisperx-subgen`. That's how it tells its files apart from Bazarr or anything else.

### Manual Queue

Use the folder queue on the main window for one-off processing without the library scanner. Add folders, click **Scan**, then **▶ Start Processing**.

## Settings

**Whisper model** — larger = more accurate but slower and more VRAM

| Model | VRAM | Speed (RTX 3090) | Notes |
|-------|------|-----------------|-------|
| tiny | ~1 GB | ~32x realtime | Poor quality |
| base | ~1.5 GB | ~16x realtime | Okay for clear audio |
| small | ~2.5 GB | ~8x realtime | Good balance |
| medium | ~5 GB | ~4x realtime | Very good |
| large-v3 | ~10 GB | ~2x realtime | Best quality |

**Compute type**
- `float16` — recommended for 8GB+ VRAM
- `int8_float16` — use if you get out-of-memory errors
- `int8` — CPU fallback, very slow

**Translation engine**
- `Argos (local, offline)` — runs fully offline after install, no rate limits, good for large batches
- `Google Translate (online)` — generally better quality but requires internet and will rate-limit on large batches

**Batch size** — number of audio chunks processed at once. Higher is faster but uses more VRAM. Start at 4–8 and increase if stable.

**Scan for lang** — language code the library scanner looks for when checking subtitle status (e.g. `no`, `en`, `sv`). Should match your target language.

## Options

- **Skip files that already have a .en.srt / .no.srt** — avoids reprocessing files that already have subtitles
- **Also save original-language .srt** — saves the raw English transcription in addition to the translated version
- **Delete existing .en.srt / .no.srt** — removes any existing subtitle files before generating new ones; useful for replacing broken Bazarr subs
- **Dry run** — scans and lists files without actually processing anything

---

## Tips

- Close Chrome, games, and other GPU-heavy apps before running large batches
- If the GPU crashes mid-batch, close and reopen the app to reset CUDA state — already-completed files will be skipped on the next run
- For a large backlog, `small` is a good starting point; upgrade to `medium` if quality isn't good enough
- The library scanner only touches the NAS when you click Scan — drives can spin down normally while the app is idle

## Troubleshooting

**"python is not recognized"**
Python is not in PATH. Reinstall Python 3.11 and check "Add Python to PATH" during setup.

**"No CUDA GPU found"**
PyTorch was installed without CUDA support. Re-run `install.bat`.

**CUDA crash mid-batch**
Lower the batch size, close background GPU apps, and re-run. The app recovers and skips already-completed files.

**Argos translation not working**
The language pack may not have installed correctly. Re-run `install.bat` — it will reinstall the pack.

**Google Translate failing**
Check your internet connection or switch to Argos in the settings.

**Subtitles out of sync**
WhisperX alignment occasionally falls back to proportional timing. Re-running the specific file usually fixes it.

**Library shows EXTERNAL for subtitles I generated earlier**
SRT files made before this tool was installed don't have the tag. Re-run those files and they'll be recognized going forward.
